        ### Implementación

        El componente Icon de MUI funciona como un envoltorio para íconos basados en fuentes (font icons) que admiten ligaduras. 

        Te permite usar nombres de íconos como contenido, heredando por defecto el color del texto circundante

        ```sh
        <Icon>star</Icon>
        ```

        Además, acepta props útiles para personalizar su apariencia dentro del tema de tu aplicación.

        ```sh
        <Icon fontSize="Small">star</Icon>
        ```
        ```sh
        <Icon color="Primary">star</Icon>
        ```

        En el contexto de nuestro sistema de diseño, este componente es clave cuando queremos:

        - Integrar íconos en contexto tipográfico (listas, botones de texto, etc.),
        - Mantener consistencia visual sin depender exclusivamente de SVGs,
        - Aplicar estilos coherentes basados en el tema (colores, tamaños, variantes).

        ### Fuente  <Badge text="Nuevo" size="medium" variant="tip" textAling="center" alignItems="center"/>

        Con esta actualización cambia el uso del componente Icon, se deja de lado y como obsoleto el uso de SVGs, para dar paso a el uso de una tipografía oficial que contiene más de 2.000 iconos.

        :::tip[¿Cómo busco los iconos?]
        Puedes tomar como referencia el sitio oficial de **[“MUI Icons”](https://mui.com/material-ui/material-icons/)**, es importante que le des el formato adecuado para que funcione, cosas que debes tener en cuenta: 
        - **No admite mayúsculas**
        - **No admite espacios**

        ¿Ahora, como debería ser el formato correcto? Ejemplo:
        - _(Sin formato)_ **AccountTreeOutlined** => _(Con formato)_ **account_tree_outlined**
        :::

        :::note[¿Qué debo tener en cuenta?]
        Cada que encuentres una mayúscula **( A )**, no solo pasala a minuscula si no agrega un **( _ )**, esto hace que como se ve en el ejemplo anterior la tipografía pueda mostrar el icono correctamente.
        :::

        ### Aspectos generales

        Los iconos deben ser usados con moderación, para apoyar ideas, conceptos o acciones. No se debe usar con fines netamente decorativos, por favor evite modificarlos o usar propiedades que no esten definidas.

        #### General

        Para funciones conocidas Ej. Eliminar _(Icono de caneca)_ use siempre que sea posible iconos ya reconocidos para estas acciones, evita el uso de iconos abstractos o destinados a usuarios muy técnicos.

        #### Uso simple

        No se recomienda el uso exesivo de iconos en una pantalla, no debe volverse pesado o un recurso común, ya que es para destacar o darle un detalle a elementos puntuales.

        #### Global

        El uso de iconos que llevan mucho tiempo representando una idea o acción son ideales para ser reconocidos y entendidos de forma rápida y eficiente; Se debe evitar reinventar iconos para funciones reconocidas o ya estandarizadas.
