      ### Implementación

      El componente Icon de MUI actúa como un contenedor para íconos basados en fuentes que utilizan ligaduras.

      Permite mostrar un ícono simplemente escribiendo su nombre como contenido, heredando por defecto el color del texto que lo rodea.

      Es ideal cuando buscas una integración visual ligera, coherente y alineada con la tipografía del sistema de diseño.

      ```sh
      <Icon>star</Icon>
      ```

      Además, el componente acepta props útiles que te permiten personalizar fácilmente su apariencia y adaptarlo al tema visual de tu aplicación.

      Puedes ajustar propiedades como <Badge text="color" variant="tip" />, <Badge text="FontSize" variant="tip" />, <Badge text="sx" variant="tip" /> o <Badge text="baseClassName" variant="tip" /> para mantener coherencia con el diseño general y asegurar que el ícono se integre de forma consistente en cualquier contexto.

      ```sh
      <Icon fontSize="Small">star</Icon>
      ```
      ```sh
      <Icon color="Primary">star</Icon>
      ```

      En el contexto de nuestro sistema de diseño, el componente Icon cumple un papel clave al:

      - Integrar íconos en el flujo tipográfico, como en listas, botones de texto o elementos informativos.
      - Mantener la coherencia visual del producto sin depender exclusivamente de íconos SVG.
      - Aplicar estilos consistentes colores, tamaños y variantes basados en el tema global de la aplicación.

      ### Fuente  <Badge text="Nuevo" size="medium" variant="tip" textAling="center" alignItems="center"/>

      Con esta actualización, el uso del componente Icon marca un cambio importante en la forma en que gestionamos los íconos dentro del sistema de diseño.

      Dejamos de lado el uso de SVGs individuales, considerándolos obsoletos en este contexto, para dar paso a una tipografía oficial de más de 2.000 íconos.

      Esta transición no solo optimiza el rendimiento y la consistencia visual, sino que también simplifica la integración y el mantenimiento de íconos en cualquier entorno o plataforma.

      :::tip[¿Cómo busco los iconos?]
      Puedes consultar la lista completa de iconos en el sitio oficial de **[“MUI Icons”](https://mui.com/material-ui/material-icons/)**
      
      Recuerda que para que funcionen correctamente en tu proyecto:
      - **Los nombres de los iconos deben estar en minúsculas.**
      - **No se deben incluir espacios en sus nombres.**

      ¿Ahora, como debería ser el formato correcto? Ejemplo:
      - _(Sin formato)_ **AccountTreeOutlined** => _(Con formato)_ **account_tree_outlined**
      :::

      :::note[¿Qué debo tener en cuenta?]
      Cada que encuentres una mayúscula **( A )**, no solo pasala a minuscula si no agrega un **( _ )**, esto hace que como se ve en el ejemplo anterior la tipografía pueda mostrar el icono correctamente.
      :::

      ### Aspectos generales

      Usa los iconos con moderación: su propósito es reforzar ideas, conceptos o acciones, no servir como elementos puramente decorativos.

      Evita modificarlos o aplicarles propiedades que no estén definidas en el sistema de diseño.

      #### General

      Para acciones comunes —como eliminar (icono de caneca)— utiliza siempre iconos reconocidos y familiares para los usuarios.

      Evita emplear iconos abstractos o excesivamente técnicos, ya que pueden generar confusión o afectar la comprensión visual.

      #### Uso simple

      Evita el uso excesivo de iconos en una misma pantalla.

      Los iconos deben ser un recurso puntual, utilizados para destacar información o reforzar acciones específicas, no como un elemento decorativo constante.
      
      #### Global

      Los iconos reconocidos y estandarizados son ideales porque transmiten una acción o idea de forma rápida y clara.

      Evita rediseñar o reinventar iconos que ya tienen un significado común: su fuerza está en la familiaridad y el reconocimiento inmediato.

      ### Contraste

      Para garantizar la accesibilidad, asegúrate de que los iconos cumplan con un contraste **mínimo AAA** en todos los casos.

      Solo en situaciones justificadas (como ajustes de peso visual), se permite un contraste **AA como nivel mínimo aceptable.**

<script is:inline>
{`
(function() {
    function initCopiarIcono() {
        const items = document.querySelectorAll('.icon-item');
        items.forEach((item) => {
            item.addEventListener('click', () => {
                const colorCode = item.dataset.color;
                navigator.clipboard.writeText(colorCode);
                item.classList.add('copied');
                setTimeout(() => item.classList.remove('copied'), 800);
            });
        });
    }
    
    if (document.readyState === 'loading') {
        document.addEventListener('DOMContentLoaded', initCopiarIcono);
    } else {
        initCopiarIcono();
    }
    
    document.addEventListener('astro:page-load', initCopiarIcono);
})();
`}
</script>
